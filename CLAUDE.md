# Working rules

## Scope

- Ne lis pas tout le repo sans nécessité.
- Commence par localiser précisément les fichiers utiles.
- Si la tâche est floue, explore au minimum avant de proposer un plan court.

## Editing

- Modifie le moins de fichiers possible.
- Réutilise l’existant avant de créer du nouveau code.
- N’effectue pas de refactor large sauf demande explicite.

## Verification

- Lance uniquement les tests/lint liés aux fichiers touchés.
- Si un test complet est inutile, préfère le test ciblé le plus petit possible.
- Résume toujours la vérification en 5 lignes max.

## Output

- Réponse finale brève.
- Donne : fichiers modifiés, raison, commande de vérification, résultat.

## Compact instructions

- Quand tu compactes, conserve uniquement :
  - fichiers modifiés
  - commandes de test utiles
  - erreurs importantes
  - décision finale
